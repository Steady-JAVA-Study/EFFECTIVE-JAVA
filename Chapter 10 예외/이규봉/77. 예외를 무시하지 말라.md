# 아이템 77. 예외를 무시하지 말라

## catch 블록을 비워두지 말자
* 예외는 문제 상황에 잘 대처하기 위해 존재한다.
* `try-catch` 사용 시 `catch` 블록을 비워두면 예외가 존재할 이유가 없어진다.

```java
try {
        ...
} catch (Exception e) {
    // 비워두지 말자
}
```

<br>

## 예외를 무시해도 될 때
* `FileInputStream` 을 닫을 때와 같을 때는 예외를 무시해도 괜찮다.
* 입력 전용 스트림이므로 파일의 상태를 변경하지 않았으니 복구할 것이 없고, 스트림을 닫는다는 것은 필요한 정보는 이미 다 읽었다는 뜻이니 남은 작업을 중단할 이유도 없다.
* 하지만, 예외를 무시하기로 했다면 왜 무시했는지 주석으로 남기고 변수명도 `ignored` 로 바꿔놓도록 하자.

```java
try {
        ...
} catch(Exception igonored) {
    // 왜 무시?
}
```

<br>

## 정리
* 예외를 무시하지 말자
* 각 예외에 대한 적절한 대처가 필요함
